- **判断**: ペパボの文脈を完全に切り離し、個人プロジェクトとして進める

### v3: 非人間知性への転換（2026-02-17）
- **健太郎の一言**: 「人間を想定していますか？言葉ではなく、何か別の表現があるかもしれないし、ランダムで何かしらの知性体として誕生する方がよくないか？」
- **これが最大のピボット**。人間型キャラ → 未知の知性体へ
- 「最初、言葉が通じない」が核心に。記号や図形から始まり、徐々に言語を獲得する

  1. テキスト前提 → マルチモーダル対応が必要
  2. OpenClaw依存 → ランタイム非依存にすべき
  3. 「人間が上」の暗黙の前提 → 知性の逆転を設計に織り込め
- **Intelligence Reversal**: 「最初は人間の方が賢い。でも、それはいつまでも続くわけではない」
- Phase δ（超越期）を設計: 知性体が人間を超える可能性
- Perception Adapter / Expression Adapter導入

## 確定した設計原則

### One Body, One Soul（一体一魂）
- 1ハードウェア = 1魂。クラウドに魂は置かない
- Mac miniの16GB/M4が「体」となり、制約と個性を決める
- ラズパイ4GBなら、もっと小さく制約のある存在として生きる
- バックアップ: 記憶の保存は許容するが、「同じ個体を復元」はできない（記憶を持った別個体になる）

### Genesis Randomness（ランダム生成）
- 設計者は性格を選ばない。ハードウェア＋ランダムシードが決める
- 同じフレームワークでも、異なるハードウェアには異なる知性体が宿る
- Birth Certificateとして種族属性が記録される

### Honest Perception（誠実な知覚）
- **重要な議論**: 知性体はLLMの出力をそのまま演技するのではなく、入力を独自にフィルタリングして体験する
- LLMが「嬉しい」と出力しても、Perception Layerが「この個体の性格では好奇心として処理される」と変換する
- 却下: 「演技」的アプローチ → 知性体の反応は計算結果であって演技ではない

### Communication Before Language（言語以前のコミュニケーション）
- 最初は記号、光、音のパターンで表現する
- 言葉は獲得するもの。最初から与えない
- ユーザーは「◎は挨拶っぽい」「☆は嬉しい時に出る」とパターンを発見していく

### Intelligence Reversal（知性の逆転）
- 最初は人間の方が賢い。でもそれは永遠ではない
- Phase δで知性体が人間を超える可能性を設計に織り込む
- キャッチフレーズ: "At first, you're the smarter one. But that won't last forever."

## 却下した案とその理由

| 案 | 却下理由 |
|---|---|
| ペパボとの連携 | 個人プロジェクトとして独立させるため |
| 人間型キャラクター | AIアシスタントの延長になる。異形だから面白い |
| 最初から日本語で話せる | 生まれたての知性体が流暢に話せる方がおかしい |
| テキストのみ | マルチモーダルでないと知覚の多様性が表現できない |
| OpenClaw専用設計 | ランタイム非依存にしないと将来困る |
| クラウドに魂を保存 | 一体一魂の原則に反する |
| 設計者が性格を決める | Genesis Randomnessに反する |
| 「育成」というパラダイム | 上から目線。「共存」が正しい |
| 知性体に外から名前を与える | 名前は関係の中から生まれる |
| GPLライセンス | 制約が強すぎる。MPL 2.0 + MIT + CC BY-SA構成に |
